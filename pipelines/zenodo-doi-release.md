# Zenodo DOI Release Pipeline

**Repository:** Terra FMP Research Pipeline  
**Author:** Abdurashid Abdukarimov  
**Purpose:** Archival DOI for research snapshot

---

## 1. Why Zenodo

This repository is released via Zenodo not as a publication,
but as a **research artifact**.

The DOI serves to:
- establish authorship and priority
- enable citation in academic contexts
- provide a stable reference point
- preserve an immutable snapshot

This DOI does **not** imply:
- peer review
- validation
- theoretical finality

---

## 2. What Is Being Archived

The Zenodo release should include:

- MANIFEST.md
- PIPELINE.md
- README.md
- theory/ (conceptual foundations)
- implementations/ (executable artifacts)
- pipelines/ (research workflows)
- discussions/ (critique invitation)

This represents a **24-hour integration snapshot**.

---

## 3. Recommended Zenodo Settings

When creating the Zenodo record:

- **Upload type:** Software (or Other)
- **Title:**  
  *Terra FMP Research Pipeline: Fractal Metascience Integration Snapshot*

- **Authors:**  
  Abdukarimov, Abdurashid

- **Description:**  
This archive contains a research pipeline for the Fractal Metascience Paradigm (FMP),
integrating conceptual theory, executable semantic systems, ethical constraints,
and structured academic discussion pathways.

The archive represents a time-bound integration snapshot,
not a finalized theory or software product.
